### 网页的布局
> 和html的主要区别是语义化，结构清晰，可读性强
#### (1)、容易被忽视的标签
> 原因:插件的滥用，丧失原生能力
```
<dl>
   <dt>计算机</dt>
   <dd>用来计算的仪器 ... ...</dd>
   <dt>显示器</dt>
   <dd>以视觉方式显示信息的装置 ... ...</dd>
</dl>
```
```
<ol>
    <li></li>
    <li></li>
    <li></li>
    <li></li>
</ol>
```
```
<ul>
   <li></li>
</ul>
```

##### （2）、布局上的性能优化
* 尽量不用id,id一般写js的时候用

##### （3）、兼容性问题
* 新的标签在某些版本浏览器下不兼容
> 解决办法:在css里面把新标签直接变成块级标签
```
header,
nav,
section,
article,
footer{
  display:block
}

//仅适用于html5新标签
```
